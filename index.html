Fase 1: Ingesta e Indexación Semántica [01]

Procesamiento del Prompt por el API Gateway de Google Cloud:

  • Tokenización de Entrada: El prompt de texto estructurado se divide en tokens y se procesa mediante un modelo Vertex AI Text Embedding.
  • Mapeo Vectorial: La cadena lingüística se convierte en vectores numéricos de alta dimensionalidad que representan conceptos abstractos.
En palabras simples: La computadora lee las palabras del prompt que escribiste y las convierte en una lista de números que representan la idea principal del texto.

Fase 2: Orquestación y Búsqueda Vectorial [02]

Recuperación de Activos con Arquitectura RAG en Tiempo Real:

  • Vertex AI Vector Search: Realiza una búsqueda de vecinos más cercanos (KNN) comparando el vector del prompt contra los metadatos de la base de datos indexada en la nube.
  • Resolución Asíncrona: Identifica de forma exacta e instantánea el ID del recurso de video óptimo correspondiente al flujo de la decisión lógica.
En palabras simples: El sistema compara los números de tu prompt con la base de datos en la nube para buscar y encontrar exactamente el video que mejor se parezca a lo que pediste.

Fase 3: Despliegue Dinámico en Cloud Storage [03]

Entrega Transmisionada del Objeto hacia la Capa del Cliente:

  • Generación de URI: El backend resuelve el puntero hacia el CDN de Google Cloud Storage (o Google Drive API de streaming seguro).
  • Inyección Dinámica al DOM: Se genera la carga asíncrona del stream en el buffer binario del reproductor web mediante peticiones XMLHttpRequest controladas por JavaScript nativo.
En palabras simples: El sistema toma el archivo de video de internet y lo introduce directamente en la pantalla de la página web sin necesidad de recargar todo el sitio.

En palabras simples: Hicimos este proyecto para demostrar cómo una página web puede conectarse a un cerebro de Inteligencia Artificial en la nube y cambiar los videos y la información de manera inmediata cuando el usuario presiona un botón, creando una aplicación interactiva fluida.

Análisis del Flujo: Nodo de Introducción

Búsqueda Semántica en Vertex AI: El prompt ingresado pasa por un filtro transaccional donde se aíslan entidades clave ("Inca Chasqui", "Cusco", "Quipu"). La base de datos vectorial de Google ejecuta un algoritmo de Similitud de Coseno para recuperar el nodo multimedia raíz con un índice de confianza del 98.4%.

Manejo de Transmisión Asíncrona: Este objeto binario se almacena temporalmente y se sincroniza con canales de audio independientes en el navegador del cliente. La API del DOM controla la velocidad del flujo de fotogramas de forma paralela al procesamiento lógico de la interfaz.

En palabras simples: La IA buscó conceptos como "Cusco" y "Chasqui" en internet, seleccionó el video de introducción que calza con esa descripción y preparó la música de fondo para que empiece a sonar sin cortes.
Análisis UX: La carga automática de este bloque de datos expone de forma directa la arquitectura de almacenamiento subyacente sin sobrecargar cognitivamente al usuario.
